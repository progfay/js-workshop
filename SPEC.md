# JavaScript研修用 演習アプリ 仕様書

## 1. 概要・目的

ブラウザ上で完結する、JavaScript研修用の自習・演習ウェブアプリケーション。
受講者がブラウザ内のエディタにコードを書き、その場で実行・自動採点できる。

- **利用シーン**: 受講者が自習で演習問題を解く
- **想定受講者**: プログラミング初心者 〜 他言語経験のあるJS初心者が混在
- **対象コード**: ECMAScript仕様の範囲内(DOM/Node固有APIは対象外)

## 2. 動作形態

- 完全クライアント完結。サーバ不要、静的ホスティングのみで動作する。
- 進捗・受講者の書いたコードは localStorage に保存する。

## 3. 技術スタック

| 項目 | 採用技術 |
| --- | --- |
| ビルドツール | Vite |
| UIフレームワーク | React |
| コードエディタ | CodeMirror |
| コード実行環境 | quickjs-emscripten(QuickJSをWASMで実行) |
| テーマ | ライト / ダーク両対応 |

- コード実行はQuickJS(WASM)で隔離して行い、**無限ループ対策のタイムアウト**を設ける。
- 問題ファイルはビルド時に同梱(コードリポジトリに配置)し、`import.meta.glob` 等で取り込む。

## 4. 問題の構成

### 4.1 ディレクトリ構造

1問を1ディレクトリとし、役割ごとにファイルを分割する。

```
problems/
  <problem-id>/
    metadata.json      # メタデータ
    problem.md         # 問題文(Markdown)
    solution.md        # 正解時に表示する解説(Markdown)
    template.js        # スターターコード
    test/
      case-01.js       # テストケース1(アサーション)
      case-02.js       # テストケース2
      ...              # N個
```

### 4.2 metadata.json

問題自体のメタデータを記述する。

```json
{
  "id": "array-sum",
  "title": "配列の合計を求める"
}
```

- **category / order(カテゴリ・表示順)** は問題ファイル側には持たせず、**問題読み込み側(アプリ側のローダ)が指定する実装**とする。
- `functionName` / `signature` は持たない。受講者が何を実装すべきかは **problem.md の本文から読み取る**。

### 4.3 problem.md

問題文をMarkdownで記述する(コードブロック等に対応)。
受講者が実装すべき関数名・仕様は、この本文で説明する。

### 4.4 solution.md

正解(全テスト通過)時に表示する解説をMarkdownで記述する。

### 4.5 template.js

エディタの初期表示に使うスターターコード。問題ごとに用意する。

### 4.6 test/case-NN.js(テストケース)

各テストケースは**アサーション関数をexportするJSファイル**とする。

- 受講者にも公開する(入出力をすべて見せる方針)。見せ方はUI側で整形して表示する。
- 採点は **throwの有無のみ** で判定する。
  - throwしなければ成功、throwすれば失敗。
- 失敗時の表示は、**throwされたErrorのmessageをそのまま表示する**。
  - 期待値・実際の値などはcase.jsの作者がmessageに含めて記述する(アプリはmessageの内容に関与しない)。
- console.logはアサーションロジックでは使用しない。

## 5. コード実行・採点モデル

### 5.1 実行単位

- **テストケースごとに個別実行する。**
- 各実行は「受講者コード + 当該 case.js」をconcatして1回の実行とし、テストケース数N回ぶん実行する。

### 5.2 採点ルール

- 各テストケースは throw の有無で合否を判定。
- **全テストケース通過時のみ「正解」扱い**(部分点なし)。
- 個別の合否は一覧で表示する。

### 5.3 console.log の扱い

- console.log は **受講者のデバッグ用機能**として提供する。
- 採点には一切使わない。
- 受講者コード内のlog出力を画面に表示する(デバッグ用)。

## 6. 実行結果の表示

実行後、以下を表示する。

1. **採点結果**: 全テストケースの合否を一覧表示。
2. **エラー・例外メッセージ**: 失敗したケースは throwされたErrorのmessageを表示。
3. **console.log出力**: 受講者がデバッグ目的で出力した内容を表示。

> ※ 当初検討した「期待値 vs 実際の値の差分の自動抽出・表示」は行わない。
>   失敗時の情報はすべてErrorのmessage経由で表示する。

## 7. ヒント・解答

- ヒント機能は**なし**。
- 解答(solution.md)は**全テスト通過(正解)時に表示**する。

## 8. 問題一覧・ナビゲーション

- 複数カテゴリで問題を分類して一覧表示する。
- カテゴリ・表示順はローダ側で指定する(4.2参照)。

## 9. 保存(localStorage)

- 受講者が書いたコードを保存する。
- 進捗(どの問題を正解したか等)を保存する。

## 10. テストケース(case.js)の記述例(参考)

> 実装詳細は別途確定するが、想定するイメージを示す。

```js
// test/case-01.js
// 受講者コードと concat されて実行される想定。
// 受講者は problem.md の指示に従い sumArray を定義している前提。

const actual = sumArray([1, 2, 3]);
const expected = 6;
if (actual !== expected) {
  throw new Error(`sumArray([1,2,3]) は ${expected} を期待しましたが ${actual} が返りました`);
}
```

- 失敗時、上記 message がそのままUIに表示される。
